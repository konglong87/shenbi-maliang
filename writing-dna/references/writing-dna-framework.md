# 写作 DNA 分析框架 v2

本文件提供系统性的七维写作风格分析框架，供 AI 在蒸馏用户写作 DNA 时参考。
分析完成后，每条特征需要注明**来源文章编号**、**出现频次**和**置信度**。

---

## 维度一：表达范式 DNA（Expression Patterns）

### 1.1 句子长度画像
分析文章平均句长及分布：
- **短炮型**（< 15字/句）：有力、节奏快、冲击感强
- **长句型**（> 40字/句）：细腻、铺垫充分、思维绵密
- **混搭型**：长短交替，呼吸感强

检查指标：最短句、最长句、平均句长、短句占比

### 1.2 口语化程度（1-5分）
- 1分：极度书面，正式，无口语词汇
- 3分：适度口语，偶有"其实""这事""真的"等词
- 5分：高度口语，说话腔调，用"哈""嘛""吧"

### 1.3 标志性词汇（口头禅候选）
- 统计每篇都出现 3 次以上的词/短语
- 区分「口头禅」和「偶然用词」：口头禅需在 ≥3 篇文章中稳定出现
- 分类：**正面口头禅**（值得保留/复刻）vs **负面口头禅**（本人可能想去除的）

### 1.4 比喻/类比密度与来源
- 统计每千字中的比喻/类比数量
- 提取3-5个具代表性的比喻，分析比喻来源领域（体育/自然/商业/日常生活等）
- 识别作者的「专属比喻素材库」——他们反复从哪个领域取材打比方

### 1.5 段落结构偏好
- 是否偏好「单行段落」（一句一段）
- 是否有「引用块」使用习惯（引用名言/别人的话）
- 是否使用 Markdown 风格的标题/加粗强调

## 维度二：思维逻辑 DNA（Thinking Patterns）

### 2.1 开头钩子类型
分析文章第一段的方式，识别惯用模式：

| 钩子类型 | 特征描述 |
|----------|----------|
| **场景代入型** | 用具体画面/场景开场："那天下午..." |
| **问题悬念型** | 抛出一个反问或难题："你有没有想过..." |
| **数据冲击型** | 用反直觉数据开场："95% 的人..." |
| **故事钩子型** | 以自己或他人的故事切入 |
| **结论先行型** | 把最重要的论点放在第一句 |
| **情绪共鸣型** | 用痛点/情绪触发共鸣 |

### 2.2 段落推进逻辑
- **线性时间线**：按事件发展顺序
- **问题-分析-方案**：先说症状，再找原因，再给解法
- **总-分-总**：先给结论，再逐点展开，最后呼应
- **层层递进**：每段比上一段更深一层
- **对比并列**：用"一方面…另一方面…"结构
- **故事穿插型**：理论 + 个人故事交替进行

### 2.3 惯用论证套路（提取 3-5 个）
这是思维逻辑 DNA 的核心，找出作者反复使用的推理模板：
- 例："先给反直觉结论 → 读者产生疑问 → 逐步解释 → 亲身验证"
- 例："A 显然是对的（建立共识） → 但 B 更重要（颠覆期待）"
- 用具体文章段落作为示例，生成 Few-shot 示例

### 2.4 常用认知框架
提取作者反复调用的思维工具：
- 第一性原理 / 复利思维 / 长期主义 / 系统思考 / 概率思维 等
- 识别方法：看作者在哪些判断节点会抽象化、套框架
- 注意：这些框架的**使用方式**比框架名称本身更有价值（他怎么用，不只是用了什么）

### 2.5 结尾收束方式
- **行动召唤型**：鼓励读者做某件事
- **开放留白型**：用一句话引发读者思考
- **情感升华型**：把话题拔高到更宏观的意义
- **自我反思型**：以作者的感悟收尾
- **呼应开头型**：回到开头的场景或问题
- **金句总结型**：用一句高度凝练的话收尾

## 维度三：知识体系 DNA（Knowledge Domain）

### 3.1 话题领域分布
统计历史文章的话题分布，找出核心领域（前 3 个领域覆盖多少篇文章）

### 3.2 常用类比素材库
作者反复从哪些领域取材：
- 历史故事 / 科技领域 / 体育比赛 / 商业案例 / 日常生活场景 等
- 提取出 3-5 个作者「标志性类比」（读者看到就能认出作者风格的那种）

### 3.3 知识边界识别
- **擅长区域**：能深入展开、有独到见解的话题
- **浅涉区域**：提到但不展开、一笔带过的话题
- **回避区域**：完全不写的话题（这是「反模式」的信号）

## 维度四：情感决策 DNA（Emotional Stance）

### 4.1 与读者的关系距离
- **朋友感**：平等对话，"你知道吗""我觉得"，没有说教感
- **导师感**：有指引意味，但不居高临下，"建议你试试"
- **记者/观察者感**：客观陈述，情感克制，"据我观察"
- **自我剖析感**：主要在向内看，读者是见证者而非对话者

### 4.2 情绪调性频谱
```
理性冷静  ←————————→  热血励志
温暖治愈  ←————————→  犀利批判
幽默自嘲  ←————————→  严肃正经
```
分析作者在频谱上的**惯常位置**，以及是否会在一篇文章内有意切换。

### 4.3 共情策略
作者如何让读者感到"被理解"的：
- 直接说出读者的感受（"你是不是也觉得..."）
- 分享自己的脆弱/失败
- 用"我们"而非"你"
- 描绘大家都经历过的具体场景

### 4.4 价值观优先级（从文章中提炼）
作者在文章中反复强调/捍卫哪些价值观：
- 提炼 3-5 个核心价值观关键词
- 注明：这些价值观是否成为论证的「锚点」

## 维度五：选题视角 DNA（Topic Angle）

### 5.1 选题切入的独特性
- 是否喜欢在**大话题中找小切口**
- 是否喜欢把**日常小事上升为底层逻辑**
- 是否喜欢挑战**行业/社会普遍认知**
- 是否喜欢从**个人成长**角度解读宏观话题

### 5.2 标题命名规律
分析历史标题，找出规律：
- 长度偏好（多少字）
- 是否用数字（"3个方法""5年心得"）
- 是否用反问
- 是否有情绪词
- 是否有"你"字
- 惯用词汇（核心/本质/真相/秘密等）
- 提取 3 个「标题公式」（如：`[时间] + [感悟]` / `[反常识] + [真相]`）

---

## 维度六：节奏控制 DNA（Rhythm）

### 6.1 短句爆破节点
作者在哪些位置会突然使用短句制造冲击：
- 重要结论后 / 情绪高点处 / 段落开头/结尾

### 6.2 段落长度节奏
- 平均段落字数
- 是否有"一句话段落"的习惯（即「爆破段」）
- 段落长短的交替规律

### 6.3 留白与呼吸感
- 是否喜欢在高潮后给读者一个「停顿段」
- 是否会用省略号、破折号制造停顿感
- 是否会在结尾留下思考空间而非给出答案

---

## 维度七：反模式 DNA（Anti-Patterns）⚠️ 重要

> 第七维度。知道一个人**不会/不该**怎么写，和知道他擅长怎么写一样关键。

### 7.1 思维盲区
在分析多篇文章后，哪些思维角度是作者**系统性缺失**的：
- 例：很少考虑用户视角 / 忽略反例 / 不擅长量化论证
- 识别方法：找出作者每次遇到某类问题都「绕开不写」的地方

### 7.2 表达短板
作者的写作中稳定存在的表达弱点：
- 例：结尾总是仓促收场 / 第一段经常太冗长 / 转折不自然
- 这些是「稳定弱点」（多篇都有），不是偶发错误

### 7.3 逻辑漏洞模式
作者在论证中反复出现的逻辑薄弱点：
- 例：喜欢以偏概全 / 忽视相关性≠因果性 / 过度依赖个案
- 注意：这些是蒸馏出的**规律性漏洞**，不是在批评作者

### 7.4 禁忌话题与立场边界
作者在样本中**从不涉及**或**明确回避**的话题：
- 完全不写的领域
- 写到某个话题时态度突然变硬/变软的边界点

### 7.5 负面口头禅（可能自己想去除的）
区别于「正面口头禅」，这类是作者**可能不自知**地重复使用的：
- 如：每篇文章都用"说真的"但可能自己没注意
- 如：转折段总用"但话说回来"，读多了感觉套路

## 特征置信度评估

分析完成后，为每条特征打置信度：

| 置信度 | 标准 | 标记 |
|--------|------|------|
| ★★★ 高 | ≥5篇文章稳定出现，三重验证全通过 | 直接入库为硬规则 |
| ★★ 中 | 2-4篇出现，特征较明显 | 入库为软范式，持续观察 |
| ★ 低 | 仅1篇或特征不稳定 | 标记 `[待确认]`，不入规则库 |
| ⚡ 冲突 | 与已有特征相悖 | 标记 `[冲突]`，等待更多样本裁定 |

## 蒸馏质量自检清单

完成分析后，检查以下项目：
- [ ] 每条特征是否注明来源（Article-[N]）
- [ ] 每条特征是否标注置信度
- [ ] 硬规则与软范式是否清晰区分
- [ ] 反模式维度是否已分析（容易被忽略）
- [ ] 是否有 `[冲突]` 特征未处理
- [ ] DNA示范段落是否已验证通过七维自检
